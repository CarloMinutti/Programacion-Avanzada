VI · Constructores — las formas que existen

No todas las clases nacen igual.
Hay cuatro formas de construir.

Desde una clase sin constructor hasta varios "constructores" alternos. Recorre las formas y lee qué hace cada una:

Ojo — esto confunde a quien viene de Java: Python no tiene sobrecarga de constructores. No puedes escribir dos __init__ con firmas distintas — el segundo pisa al primero. La "sobrecarga" pythónica se logra de dos formas: con argumentos default (un constructor flexible que acepta distintos números de argumentos) y con factory methods usando @classmethod (constructores alternos, cada uno con su nombre).

VII · @dataclass — cuando Python escribe por ti

Si tu clase es solo datos,
no escribas el molde a mano.

Compara la misma clase Punto escrita a mano y con @dataclass. Hacen exactamente lo mismo:

¿Qué genera exactamente? Dos métodos clave:
__repr__ y __eq__.

__repr__
Sin definir>>> p
<__main__.Punto object at 0x7f8b...>
Con __repr__ (o @dataclass)>>> p
Punto(x=3, y=5)
Es la representación oficial del objeto — la que ves en el REPL, al hacer debugging y al imprimir listas de objetos. Por convención debería poder recrear el objeto. Diferencia con __str__: __str__ es para el usuario final (amigable); __repr__ es para el desarrollador (sin ambigüedad). Si solo defines __repr__, Python lo usa para ambos.
__eq__
Sin definir>>> p1 == p2
False  # aunque x, y sean iguales
Con __eq__ (o @dataclass)>>> p1 == p2
True   # compara los datos
Define qué significa ==. Por defecto Python compara identidad (si son el mismo objeto en memoria), no contenido — por eso dos objetos idénticos dan False. Con __eq__ comparas por datos. @dataclass lo genera comparando todos los campos declarados.

VIII · ¿Cuándo usar qué?

Tres herramientas.
Cada una, su momento.

No todo necesita @property, y no todo es @dataclass. La guía rápida para decidir:

@propertyleer con lógica detrás El valor se calcula o debe protegerse al leerse El atributo es simple y público, sin lógica nombre_completo
token enmascarado
@x.settervalidar al asignar Necesitas validar o transformar antes de guardar No hay nada que validar: un atributo normal basta edad (0-120)
