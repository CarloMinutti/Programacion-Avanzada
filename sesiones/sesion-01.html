V  ·  Sobre la inteligencia artificial

Sí, van a usar IA. Y van a hacerlo aquí, delante de mí.

Van a usar ChatGPT. Van a usar Copilot. Van a usar Claude. Sin esconderse, sin pedir perdón, sin esa cara de culpa que tienen los que creen que la IA es trampa.

La IA no es trampa. La IA es una interlocutora. Sócrates tenía a sus discípulos para discutir. Aristóteles, a sus aristotélicos. Ustedes tienen un modelo de lenguaje. La diferencia: el modelo no se cansa ni se ofende — pero tampoco piensa por ustedes.

Vamos más lejos: la IA va a ser contenido del curso, no solo herramienta. Prompt engineering, agentes, MCPs, skills. Cosas que en otra clase ni se nombran — aquí las van a construir. Porque la industria ya está ahí y ustedes se gradúan pronto.

Si la IA lo hizo, tú lo defiendes.

Esa es la regla. La única. Si no pueden explicarle a su abuela — o a mí, que vengo a ser lo mismo — qué hace ese código y por qué, entonces no es su código. Es código prestado. Y el código prestado no se aprueba.

Un entregable impecable que no pueden explicar pierde puntos. Un entregable imperfecto, lleno de cicatrices, pero defendido con argumentos, es valiosísimo.

Cuatro lenguajes. Cuatro formas de ver el mundo.

Saber un solo lenguaje es como leer un solo libro y creer que ya entendiste la literatura. Cada lenguaje es una filosofía — y vamos a conocerlas todas.

01
Python
El lenguaje legible.
El pragmatismo elevado.
Como Sócrates: pregunta antes de afirmar, lee como prosa, no intimida. Te invita a pensar en voz alta. Nuestro lenguaje base.
02
Java
El lenguaje del rigor.
El estoico.
Como Aristóteles: cada cosa tiene su forma, su categoría, su lugar. Verboso, sí — pero honesto. Nada de sorpresas a las tres de la mañana.
03
JavaScript
El caos creativo.
El sofista.
Puede convencerte de cualquier cosa. Tipos débiles, fronteras flexibles, [] == ![] es true. Y aún así, gobierna la web.
04
Go
El austero.
El minimalista.
Como Diógenes: sin adornos, sin lujos. Solo lo necesario. Te obliga a manejar errores a mano. Curativo.

VIII  ·  Hello, World — cuatro filosofías

El mismo saludo. Cuatro acentos distintos.

Lean los cuatro. Pregúntense: ¿por qué decir lo mismo cuesta tantas líneas en uno y tan pocas en otro? Esa pregunta es filosofía pura — y es la primera tarea de este curso.

Pythonhello.py
print("Hola, mundo.")
# Una línea. Una intención. Una idea.
JavaHello.java
public class Hello {
  public static void main(String[] args) {
    System.out.println("Hola, mundo.");
  }
}
JavaScripthello.js
console.log("Hola, mundo.");
// Funciona en el navegador, en el servidor,
// en la tostadora. Para bien y para mal.
Gohello.go
package main

import "fmt"

func main() {
  fmt.Println("Hola, mundo.")
}

IX  ·  El ritual del commit

Sin commits, no exististe.
Sin push, nadie lo sabe.

Git no es opcional. Git es la memoria del programador — el diario que no olvida, no miente y no acepta excusas. Lo que no está en un repositorio con commits limpios, para el resto del mundo, no existe. Aprendan el ritual ahora, de una vez, y practícenlo desde el primer día.

01
Clonar / Iniciar
git clone [url]
# o desde cero:
git init
Primero, el territorio. Clonan un repo existente de GitHub o inician uno vacío en su máquina.
02
Revisar estado
git status
Antes de actuar, observen. Git les dice qué cambió, qué no está rastreado y qué ya está listo para confirmar.
03
Agregar cambios
git add .
# o un archivo:
git add hello.py
Le dicen a Git qué va en el commit. El punto ( . ) agrega todo. Con nombre, solo ese archivo.
04
Confirmar
git commit -m "mensaje"
La firma. El mensaje describe qué hicieron. "fix stuff" no es un mensaje — es una evasiva.
05
Subir
git push origin main
El mundo lo ve. Hasta aquí el commit vive solo en su máquina. Con push, vive en GitHub.
flujo completo  ·  sesión i  ·  bash
# 1. Clonar el repositorio desde GitHub $ git clone https://github.com/tu-usuario/mi-primer-repo.git Cloning into 'mi-primer-repo'... $ cd mi-primer-repo # 2. Crear los archivos hello world $ touch hello.py Hello.java # (escribe tu código en cada archivo) # 3. Ver qué cambió — Git detecta los archivos nuevos $ git status On branch main Untracked files: hello.py, Hello.java # 4. Agregar todos los cambios al área de staging $ git add . # 5. Confirmar con mensaje descriptivo $ git commit -m "Add hello world in Python and Java" [main a3f1c2d] Add hello world in Python and Java 2 files changed, 2 insertions(+) # 6. Subir al repositorio remoto en GitHub $ git push origin main Enumerating objects: 4, done. To https://github.com/tu-usuario/mi-primer-repo.git main -> main ✓
git log --oneline
Ver el historial de commits en formato compacto. La memoria de todo lo que existió.
git pull origin main
Traer cambios del repositorio remoto. Imprescindible cuando trabajan en equipo o desde dos equipos.
git diff
Ver exactamente qué cambió línea por línea. Úsenlo antes de hacer add — siempre.
git checkout -b nombre
Crear y cambiarse a una rama nueva. Cada feature, su rama. Cada rama, su historia.

El mensaje del commit es una promesa. Escríbanlo en imperativo, en inglés (es la lengua de la industria), y describan el qué, no el cómo.

En BBVA, un commit con mensaje genérico en producción puede detener code reviews enteros. No sean ese desarrollador.

  • Add hello world in Python
  • Fix null reference in auth
  • Update README with setup
  • Remove unused imports
  • fix stuff
  • changes
  • aaa
  • wip final final v3
