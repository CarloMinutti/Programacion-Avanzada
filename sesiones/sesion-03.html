III · El misterio de self

self es el "yo" del objeto.
Es el objeto hablando de sí mismo.

Cuando escribes ana.saludar(), Python pasa ana como self automáticamente. Elige un objeto y míralo:

ana
nombre = "Ana"
luis
nombre = "Luis"
sofia
nombre = "Sofía"
Llamas: ana.saludar()
↓ Python lo traduce a ↓
Estudiante.saludar(self = ana)

V · La misma clase, dos lenguajes

Python dice poco.
Java dice todo, explícito.

Misma idea, distinto carácter. Toca un concepto y mira dónde vive en cada lenguaje:

Python conciso
class Estudiante: def __init__(self, nombre, edad): self.nombre = nombre self.edad = edad def saludar(self): return f"Hola, soy {self.nombre}"
Java explícito
public class Estudiante { private String nombre; private int edad; public Estudiante(String nombre, int edad) { this.nombre = nombre; this.edad = edad; } public String saludar() { return "Hola, soy " + this.nombre; } }

VI · Getters & Setters

No dejes que nadie meta basura
en tus atributos. El setter es el guardia.

Asignar directo no valida nada: ana.edad = -5 pasa sin chistar. Un setter revisa antes de guardar. Prueba a romperlo:

def set_edad(self, valor): if 0 <= valor <= 120: self.edad = valor # ✓ else: raise ValueError("Edad inválida")

VII · Representación e identidad

Sin __str__, tu objeto se imprime feo.
Con él, se imprime humano.

El método __str__ (el toString de Java) decide qué ves al imprimir. Actívalo:

>>> print(ana)
<__main__.Estudiante object at 0x7f8b2c0a1d30>
def __str__(self): return f"Estudiante({self.nombre}, {self.edad})"
¿Dos objetos con los mismos datos son iguales?
ana1 = Estudiante("Ana", 20)
ana2 = Estudiante("Ana", 20)
ana1 == ana2 → False
Sin __eq__, Python compara su lugar en memoria. Son objetos distintos aunque tengan los mismos datos.
